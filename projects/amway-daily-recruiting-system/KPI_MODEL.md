# KPI_MODEL.md

## KPI Framework
แบ่งเป็น 5 ชั้น
1. Activity
2. Response
3. Qualification
4. Invite
5. Review / Learning

## Daily KPIs
### Input KPIs
- จำนวนคอนเทนต์ที่เผยแพร่: 1-2
- จำนวน warm outreach ใหม่: Starter 10-15 / Expanded 20-30
- จำนวน follow-up ที่ทำ: Starter 5-10 / Expanded 15-25
- จำนวน qualified conversations: 2-5
- จำนวน invites sent: 1-5

### Output KPIs
- reply rate ของ outreach
- reply rate ของ follow-up
- % conversations ที่ไปสู่ qualification
- % qualified prospects ที่รับ invite
- show-up rate ของ call/meeting

## Weekly KPIs
- new conversations ทั้งสัปดาห์
- inbound vs outbound conversation split
- leads added
- qualified prospects added
- invites sent
- calls/meetings completed
- next-step secured
- no-fit / not-now count

## Conversion Benchmarks (ใช้เป็น reference ไม่ใช่คำสัญญา)
ตัวเลขด้านล่างเป็นช่วงสำหรับการประเมินสุขภาพ pipeline เท่านั้น:
- warm opener reply rate: 20-45%
- follow-up reply rate: 15-35%
- qualified-to-invite: 30-60%
- invite acceptance: 25-50%
- show-up rate: 60-85%

หากต่ำกว่านี้ ไม่ได้แปลว่าแย่เสมอไป อาจมาจากคุณภาพรายชื่อ, positioning, timing, หรือข้อความไม่ตรงบริบท

## KPI Diagnosis Map
### ถ้า outreach เยอะแต่ reply ต่ำ
ตรวจ:
- opener generic เกินไปหรือไม่
- รายชื่อเย็นเกินไปหรือไม่
- ส่งถี่เกินไปหรือไม่
- profile / content support อ่อนเกินไปหรือไม่

### ถ้า reply ดีแต่ไม่ไป qualification
ตรวจ:
- คุยวกวน ไม่มี transition
- รีบ pitch เร็วเกิน
- ถามแต่เรื่องตัวเอง ไม่ถามบริบทเขา

### ถ้า qualification ดีแต่ invite ต่ำ
ตรวจ:
- ยังไม่ชัดว่า invite ไปเพื่ออะไร
- offer ไม่ relevant กับ pain/goal เขา
- timing ยังเร็วเกิน

### ถ้า invite ได้แต่ไม่ show
ตรวจ:
- ไม่มี pre-frame
- ไม่ยืนยันเวลา
- call ดูจริงจังเกินจนคนถอย

## Minimum viable dashboard

1. New conversations
2. Leads added
3. Replies

5. Invites sent
6. Invites accepted
7. Meetings completed
